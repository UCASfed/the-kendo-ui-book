Chapter 1. Why Kendo UI

1.1 Comprehensive set of widgets & tools for building web sites and applications

Kendo UI is first and foremost a comprehensive set of reusable user interface widgets, based on jQuery and written in JavaScript, to be used in the creation of applications written for the web platform (i.e. browsers running on computers, tablets, phones).

Kendo is divided into 4 parts.

  1. Web widgets - User interface widgets (e.g Date Picker)
  2. Data Visualization widgets (AKA DataViz) - Widgets for displaying data visually (e.g Pie Chart)
  3. Application/Framework tools - Tools for building web applications specially JavaScript applications using a Single Page Application Architecture (e.g. Views).
  4. Mobile widgets & Mobile Application/Framework tools - User interface widgets and set of application tools for building browser based applications on a phone or tablet (e.g. Tool Bar).

The totality of Kendo UI parts makes it the most complete set of JavaScript widgets and tools from one source that run in the widest range of browsers on the widest range of hand held devices and computers.

Notes
  • Many of the official Kendo UI documents (i.e. Getting Started, API documents) are divided into the above mentioned parts.
  • Mobile widgets, by design, function on laptop and desktop browsers.  This makes responsive, mobile-first UIs trivial to implement on phone and tablet devices, as well as desktop and laptop computers (desktop support for mobile specific widgets: Chrome Latest, Firefox latest, Opera 15+, Safari 5+, and IE 10+).
  • The Web and DataViz widgets are supported on all the obvious modern browsers, but  also include full support for IE7+ on Windows XP and Server 2003.
  • The DataViz widgets function on iPhones and tablet devices.
  • The Web and DataViz widgets are designed to function on both a mouse-driven and touch-driven interface, and many of the web widgets (e.g. Grid and Scheduler) offer an auto-mobile mode (aka mobile adaptive rendering).
  • Kendo UI boasts the widest reaching mobile widgets by supporting the broadest range of devices/platforms (iOS 6.0+, Android 2.3+ (In Android 2.3 dataviz widgets support only Canvas rendering mode), BlackBerry OS 10+, Windows Phone 8+, Chrome for Android latest)

1.2 The bulk of Kendo UI is free and open source

Most of Kendo UI is free (AKA Kendo UI Core), and can be found on Github under the permissive Apache 2.0 license. Historically Kendo UI Core was a commercial solution but as of April 2014 it's freely available to anyone to be used for anything. No catch! You get a commercial and enterprise grade solutions for free!

Telerik also offers a set of commercially licensed (AKA Kendo UI Professional) DataViz widgets (i.e. charts, gauges, barcode, QR code, stock chart, and treemap widgets) and the following Web widgets:

  • Diagram
  • Editor
  • Gantt
  • Grid
  • Map
  • PivotGrid
  • Scheduler
  • TreeView
  • Upload

1.3 In-depth Getting Started docs and API docs

Kendo UI provides API documentation which details options, methods, and events for each widget or tool. Additionally, above and beyond the API documentation a Getting Started overview is provided for each widget or tool. Very few commercial solutions provide such a resource, and if they do, its nowhere as in depth as what is found in Kendo UI Getting Started documents.

1.4 Detailed demos immediately runnable and editable

Each widget contains a plethora of demos (e.g. the grid offers 36+ unique demos), all of which are immediately available to be edited in an official live, online code editor. The fact that the demos are immediately editable in an official editor sets Kendo UI apart from its competitors.

1.7 Numerous options for including Kendo UI JavaScript

Kendo UI provides the most comprehensive set of options for including Kendo in a web page. You can include Kendo UI in one of the following four ways:

  1. One JavaScript file - Kendo UI can be included in a web page using a single JavaScript file
  2. Individual source files - Kendo UI source is divided into a set of individual JavaScript modules, which have dependencies on each other. These files can individually be added to a web page and the dependencies will have to be managed manually as opposed to using a dependency manager/loader like RequireJS.
  3. AMD module/dependency loader - Because each source file is an AMD module, a modular loading system such as RequireJS can be used to load and manage Kendo UI dependencies. Using a dependency loader alleviates having to manage dependencies by manually ordering inline <script> tags.
  4. One custom built file - Using the Custom Download Tool a custom version of Kendo UI can be downloaded as a single file.

This makes Kendo UI the most versatile commercial solution for developers of varying skill, creating varying types of projects, from simple web pages to complex applications.

Notes
  • Options 1-3 above can be done locally or by using an official CDN.
  • To make a custom build of Kendo UI you must create an account and be logged in to use the Custom Download tool.
